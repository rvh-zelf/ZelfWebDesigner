# Expert-style critique (optional)

Triggered by: "review", "score", "is it good?", "critique", or proactive QA on risky briefs.

## Six dimensions (0–10 each)

1. **Philosophical consistency** — does execution match the stated direction?
2. **Visual hierarchy** — can a first-time viewer parse priority in ≤3s?
3. **Craft / detail** — spacing, type rhythm, edge cases, motion quality.
4. **Functionality** — clicks, focus, readability, reduced-motion path.
5. **Novelty / memorability** — one distinctive hook vs generic trope reuse.
6. **Accessibility** — WCAG 2.2 AA compliance: semantics, contrast, keyboard, ARIA, touch targets.

## Output format

- **Overall** paragraph (non-personal: critique the artefact, not the designer).
- **Keep** — 3 bullets of what works.
- **Fix** — grouped by severity: **fatal / important / polish**.
  - Fatal accessibility failures (missing ARIA, contrast fail, keyboard trap) count as "fatal".
- **Quick wins** — top 3 changes under ~5 minutes each.
