# Workflow · questions and pacing

## Batch questions (default)

Ask **one focused round** (≤3–5 items), then **wait** for answers before building large UI.

### Typical first round

- Existing **design system / UI kit / Figma / screenshots / repo** paths?
- Primary **audience** and **job-to-be-done** (what must they understand or do)?
- **Deliverable shape**: marketing page, dashboard, deck, motion demo, app prototype, infographic?
- **Variations**: how many directions? Which axes (colour, density, layout, motion)?
- **Interactivity level**: static, Tweaks, full flow demo?
- **Brand**: named client/product? If yes, trigger core asset protocol in `SKILL.md`.

### Deck-specific (see `slide-decks.md`)

- Canonical **HTML deck** first — confirm whether **PDF** or **editable PPTX** is needed (PPTX constrains HTML early).

### Tracking

Paste a short markdown checklist in the thread and tick mentally before claiming "done".
